Infinite, Persistent World

Chunks stream in and out as you move across the landscape

The world is broken into "chunks" - 16x16x128 block groups that stream in and out as the player moves around. At all times, the game manages these chunks in a radius around the player. When the player moves, chunks that are flushed from memory are written out to disk - this ensures that when the player comes back, anything that they built still stays around.

Chunks are generated with their heights calculated through a seeded 2D perlin noise function - a randomized, deterministic algorithm that ensures smooth gradients across the world plane.

Biomes

A separately seeded 2D perlin noise function is used to determine world biomes. These perlin grids are used as temperature and humidity in combination with column height. The base perlin noise function returns values between -1 and 1, so we use 0 as our determining factor on "hot", "cold", "wet", and "dry".

Water and ice are determined by column heights being less than an arbitrarily defined "sea level".

Rain and Snow Effects

Snow on top of a mountain

If a column is marked as MOUNTAIN (It's column height exceeds a certain threshold), snow spawns in that column. If a column is marked as PLAINS (It has high humidity and temperature), rain spawns in that column.

For each column marked as snow or rain, 4 weather "sheets" (2D oriented quads) spawn and move downwards toward the ground. Each frame, the weather sheet checks to see if it's bottom is intersecting a block, and moves back up to the top of the column.

Clouds (3D axis-aligned rectangular prisms) are spawned randomly above snow and rain columns and move horizontally across the terrain.

Lighting

Block faces get darker the further down you dig

Light propagates using internal, arbitrarily defined light values of blocks that light the faces of surrounding blocks. Every block starts with an internal light value of 1, so that if no light is coming into a concealed area, it will still remain partially visible.

When a chunk is first generated, the terrain generator descends each column and marks each air tile as sky. It then reiterates through each column, checks surrounding horizontal neighbors, and "dirties" them if they're not air tiles. Each frame, the algorithm goes through all blocks that need their lighting recalculated and adjusts their internal lighting values to correspond correctly to their neighbors.

Blocks only need their lighting recalculated when neighboring blocks are added or removed, ensuring that lighting is not calculated unless it needs to be.
